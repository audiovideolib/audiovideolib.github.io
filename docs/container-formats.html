Container formats

Walkers in AudioVideoLib.IO. Each implements IMediaContainer and is auto-detected by MediaContainers.ReadStream(stream). Pick a walker below for its on-disk shape, the public API surface, and any quirks.

Walker Container Surfaces
MpaStream MPEG-1 / MPEG-2 / MPEG-2.5 audio (Layer I / II / III) frame-by-frame MpaFrame + Xing / LAME / VBRI VBR header
FlacStream FLAC metadata blocks (STREAMINFO, VORBIS_COMMENT, PICTURE, …) + frames
RiffStream RIFF / WAV / RIFX fmt / data + LIST INFO / id3 / bext / iXML side-channels
AiffStream AIFF / AIFF-C COMM / SSND + NAME / AUTH / ANNO / COMT text chunks
OggStream OGG page-by-page; codec (vorbis / opus), channels, sample rate
Mp4Stream MP4 / M4A top-level boxes + moov.udta.meta.ilst iTunes metadata
AsfStream ASF / WMA / WMV Header Object children + aggregated metadata tag
MatroskaStream Matroska / WebM EBML header + first Segment (Info, Tags)
DsfStream / DffStream DSD audio (Sony DSF, Philips DFF) DSD / fmt / data chunks + embedded ID3v2

Container-embedded metadata (MP4 ilst, ASF, Matroska tags, WAV side channels, AIFF text, DSF/DFF embedded ID3v2) is documented under Tag formats. The walkers above describe how the container is parsed and which properties carry the metadata models.
