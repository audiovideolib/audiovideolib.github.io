Remarks

Splice rewriters (MP4, ASF, Matroska, DSF, DFF) need to copy unchanged byte ranges from the source at write time. Holding the entire input in memory does not scale to multi-GB files; ISourceReader abstracts the random-access read so the same walker can run against either an in-memory buffer (MemorySourceReader) or a seekable file stream (StreamSourceReader).

Implementations are not required to be thread-safe — a walker should hold a single reader and use it sequentially. Callers must keep the underlying source (file stream, memory buffer) alive until the walker is done with it; IDisposable is implemented so implementations can release their own resources, but the caller's source is not owned.
