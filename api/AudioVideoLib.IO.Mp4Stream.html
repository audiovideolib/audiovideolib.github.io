Remarks

The walker keeps an ISourceReader reference rather than buffering the file in memory; WriteTo(Stream) streams unchanged regions directly from source to destination, so editing iTunes metadata in a multi-GB MP4 is bounded by the metadata size, not the file size.

Box sizes are bound-checked against the parent; size==0 ("to end of file") and size==1 ("64-bit extended size in the next 8 bytes") are both supported. Recursion is bounded by an explicit depth cap to avoid stack blow-up on malicious input.

Callers must keep the source System.IO.Stream passed to ReadStream(Stream) alive until WriteTo(Stream) / ToByteArray finishes.
