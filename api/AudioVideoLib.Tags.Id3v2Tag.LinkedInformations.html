LinkedInformations

Gets or sets a list of linked information.

public Id3v2FrameCollection<Id3v2LinkedInformationFrame> LinkedInformations { get; set; }

Property Value

Id3v2FrameCollection<Id3v2LinkedInformationFrame>

A list of linked information.

Remarks

To keep space waste as low as possible this property may be used to link information from another Id3v2 tag that might reside in another audio file or alone in a binary file. It is recommended that this method is only used when the files are stored on a CD-ROM or other circumstances when the risk of file separation is low. The frame contains a frame identifier, which is the frame that should be linked into this tag, a URL [URL] field, where a reference to the file where the frame is given, and additional ID data, if needed. Data should be retrieved from the first tag found in the file to which this link points.

There may be more than one Id3v2LinkedInformationFrame frame in an Id3v2Tag, but only one with the same FrameIdentifier, Url and AdditionalIdData.

A linked frame is to be considered as part of the Id3v2Tag and has the same restrictions as if it was a physical part of the tag (i.e. only one Id3v2ReverbFrame frame allowed, whether it's linked or not).

Frames that may be linked and need no additional data are Id3v2InvolvedPeopleListFrame, Id3v2MusicCdIdentifierFrame, Id3v2EventTimingCodesFrame, Id3v2SyncedTempoCodesFrame, Id3v2RelativeVolumeAdjustmentFrame, Id3v2EqualisationFrame, Id3v2ReverbFrame, Id3v2RecommendedBufferSizeFrame, the Id3v2TextFrames and the Id3v2UrlLinkFrame frames.

The Id3v2UserDefinedTextInformationFrame, Id3v2AttachedPictureFrame, Id3v2GeneralEncapsulatedObjectFrame, Id3v2EncryptedMetaFrame and Id3v2AudioEncryptionFrame frames may be linked with the content descriptor as AdditionalIdData.

The Id3v2CommentFrame, Id3v2SynchronizedLyricsFrame and Id3v2UnsynchronizedLyricsFrame frames may be linked with three bytes of language descriptor directly followed by a content descriptor as AdditionalIdData.
