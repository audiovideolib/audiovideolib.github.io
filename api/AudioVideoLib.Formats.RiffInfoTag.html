Remarks

The standard four-character codes (INAM, IART, IPRD, ICRD, ICMT, IGNR, ITRK, IENG, ISFT, ICOP) are surfaced as named properties. All other sub-chunks fall through to Items.

Properties

Artist

Gets the artist (IART).

Comment

Gets the comment (ICMT).

Copyright

Gets the copyright (ICOP).

CreationDate

Gets the creation date (ICRD).

Engineer

Gets the engineer (IENG).

Genre

Gets the genre (IGNR).

Items

Gets the full set of parsed key/value items, keyed by four-character chunk id.

Product

Gets the product / album (IPRD).

Software

Gets the software (ISFT).

Title

Gets the title (INAM).

Track

Gets the track number (ITRK).

Methods

FromListPayload(byte[])

Parses a full WAV LIST chunk payload — including the leading 4-byte form-type — into a tag, or returns null if the form-type is not INFO or input is malformed.

Parse(ReadOnlySpan<byte>)

Parses a LIST chunk payload (form-type INFO already consumed) into a tag, or returns null for malformed input.

SetItem(string, string?)

Sets or replaces a single item by its four-character id. Use null to remove.

ToByteArray()

Serialises the tag's sub-chunks (without the enclosing LIST header or form-type). Sub-chunk values are written as null-terminated ASCII and word-aligned.

ToListChunkBytes()

Serialises the tag wrapped in a full LIST INFO chunk (8-byte header + form-type + items).
