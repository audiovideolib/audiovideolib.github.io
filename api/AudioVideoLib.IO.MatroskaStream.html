Remarks

Supports round-trip serialisation: the source bytes are kept reachable via an ISourceReader reference (no full-file copy in memory), and a new Tags element is spliced in (or appended) at WriteTo(Stream) time by streaming unchanged regions directly from the source. This is what makes editing a 40 GB MKV viable: peak memory stays in the low MB range regardless of file size.

Callers must keep the underlying source (the System.IO.Stream passed to ReadStream(Stream)) alive until WriteTo(Stream) / ToByteArray finishes — the walker reads from it on demand.
