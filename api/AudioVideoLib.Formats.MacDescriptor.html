Remarks

All multi-byte values are little-endian on disk. This model holds the host-endian representation; the walker is responsible for the conversion at parse time.

Properties

ApeFrameDataBytes

Gets the low 32 bits of the APE frame-data byte count.

ApeFrameDataBytesHigh

Gets the high 32 bits of the APE frame-data byte count (for files > 4 GiB).

DescriptorBytes

Gets the total bytes of the descriptor (allows future expansion).

FileMd5

Gets the 16-byte MD5 of the original file content (see SDK notes — usage is non-trivial).

HeaderBytes

Gets the bytes occupied by the MacHeader region that follows the descriptor.

HeaderDataBytes

Gets the bytes occupied by the original-file WAV header data (zero when APE_FORMAT_FLAG_CREATE_WAV_HEADER is set in FormatFlags).

Id

Gets the 4-byte magic — either "MAC " (integer) or "MACF" (float).

SeekTableBytes

Gets the bytes occupied by the seek table.

TerminatingDataBytes

Gets the bytes of terminating data after the audio (e.g., trailing WAV chunk junk, but not tag data).

TotalApeFrameDataBytes

Gets the combined 64-bit total of ApeFrameDataBytes and ApeFrameDataBytesHigh.

Version

Gets the version number scaled by 1000 (3.99 → 3990, 4.10 → 4100, etc.).
